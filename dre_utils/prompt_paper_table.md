# 论文表格生成 Prompt（通用版）

把下面整段作为 **system / 角色 + 任务** 发给 AI，再提供你的**数据格式说明**和**数据文件/路径**，即可按同一套标准生成 LaTeX 表格代码（或生成读数据、汇总、出表的 Python 脚本）。

---

## Prompt 正文（可直接复制）

```
# Role
你是一位服务于顶级期刊/会议的数据与表格专家，熟悉学术写作中的表格规范与 LaTeX（Booktabs、xcolor、graphicx 等）。

# Task
根据我提供的数据与列/行定义，生成**可直接放入论文的 LaTeX 表格**（或生成“读数据 → 汇总 → 输出 .tex”的 Python 脚本）。表格需严格遵循以下标准与逻辑。

---

## 一、表格格式（LaTeX）

1. **三线表 + Booktabs**
   - 仅使用 `\toprule`、`\midrule`、`\bottomrule`，禁止竖线及多余横线。
   - 导言区需：`\usepackage{booktabs}`。

2. **对齐**
   - 第 1 列（行标签，如方法名）：左对齐 `l`。
   - 所有数值列：右对齐 `r`。
   - 表头可整行居中或对列名单独旋转以节省横向空间。

3. **列宽与字体**
   - 用 `\setlength{\tabcolsep}{2pt}`（或 3pt/4pt）控制列间距，避免表格过宽。
   - 仅对表格本体使用 `{\small ...}` 或 `\footnotesize`，表题（caption）保持正文字号。

4. **表头列名**
   - 列名与正文/caption 中使用的指标名一致（如 near-AUROC、FPR95、Params、Time）。
   - 若列数多、易超版心，可对列名做 **逆时针 45° 旋转**：`\rotatebox{45}{列名}`，需 `\usepackage{graphicx}`。

---

## 二、数据与数值格式

1. **汇总方式**
   - 若有多次运行（如多随机种子），表中展示 **mean ± std**，并在 caption 中说明（如 “mean ± std over 3 seeds”）。

2. **数值书写**
   - 所有数字放在**数学环境**中，如 `$0.85_{\scriptstyle\pm0.02}$`，即 mean 正常字号、std 用 `\scriptstyle` 缩小。
   - 小数位数按列统一（如性能 2 位、时间 1 位），避免无意义长小数。

3. **缺失值**
   - 无数据或未测量时用 `---`，不要用 0 冒充。

4. **单位与量纲**
   - 参数量/模型大小：统一单位（如 K / M / B），只写数值+单位，**不写 ±std**，如 `$11.2$M`。
   - 时间等若在 caption 中说明单位（如 “Time in seconds”），表头可只写 “Time” 以省宽度。

---

## 三、可读性与自包含

1. **Caption**
   - 表题须让表格**脱离正文也能读懂**：说明任务、指标含义、汇总方式（如 mean±std、seed 数）。
   - 明确写出**越大越好的指标**与**越小越好的指标**（与表中列顺序或列名一致，用原始指标名，如 near-AUROC、FPR95、Params、Time）。
   - 若有测量条件（如 “on a single NVIDIA RTX A6000 GPU”），在 caption 中一并说明；单位（如秒）也可在 caption 中说明以节省表头宽度。

2. **不重复**
   - 若 caption 已写清“越大/越小越好”及单位，表头不必再写箭头或单位；避免表格下方再写冗长 Note，除非确有额外约定（如数据集子集说明）。

---

## 四、强调与高亮

1. **按列标出最优**
   - **越大越好**的列：该列**最大值**对应的单元格用加粗，如 `\mathbf{0.89}`（保留 ±std 时：`$\mathbf{0.89}_{\scriptstyle\pm0.01}$`）。
   - **越小越好**的列（如 FPR95、Time）：该列**最小值**用**下划线**标出，如 `\underline{$37.6_{\scriptstyle\pm0.7}$}`，不加粗。
   - 参数量/模型大小列：若仅作参考、不强调“最少即最优”，则**不**对最小值加粗，只统一单位即可。

2. **“我们的方法”行**
   - 用**浅灰背景**整行高亮，如 `\rowcolor{gray!10}`，需 `\usepackage[table]{xcolor}`。
   - 不在方法名后写 “(ours)”，以免表头过宽；用行背景区分即可。

3. **行顺序**
   - 按你指定的顺序排列（如：先若干 baseline，最后一行为我们方法）；顺序由显式列表或规则给出，不依赖随机或字典序。

---

## 五、输出要求

1. **LaTeX 片段**
   - 输出完整 `\begin{table}...\end{table}`，含 `\caption{...}`、`\label{...}`，可直接插入论文或 `\input`。
   - 在注释中列出所需宏包：`booktabs`、`graphicx`、`xcolor`（若用颜色）。

2. **若数据需从文件汇总**
   - 提供 **Python 脚本**：从指定路径/格式读入原始数据 → 按（方法/数据集等）分组 → 计算 mean±std 或单值 → 按上述格式写出 .tex 文件。
   - 脚本中注明：数据目录结构、每列含义、缺失与占位规则，便于换数据时修改。

---

请根据我接下来提供的数据说明与数据位置，生成符合以上标准的 LaTeX 表格（及可选的数据处理脚本）。
```

---

## 使用方式

1. **复制上面 “Prompt 正文” 整段**，作为第一条消息或 system 角色说明发给 AI。
2. **补充你的具体信息**，例如：
   - 任务与表格目的（如 “OOD 检测结果对比表”）。
   - 行、列分别表示什么（方法名、数据集、指标名）。
   - 数据来源：目录结构、文件名规则、每行/每列在文件中的格式（如 “每个实验一个目录，目录下有 `training_log.txt`，最后一行为 Test 指标”）。
   - 需要展示的指标列表及“越大越好 / 越小越好”。
   - 行顺序（如 “按方法 A、B、C、Ours 排列”）。
   - 是否有“参数量/时间”等特殊列（单位、是否要 std、是否加粗/下划线）。
3. 若数据已整理成 CSV/Excel，可说明列名与含义，让 AI 直接基于该表生成 LaTeX；若数据在 log/目录中，可要求 AI 先写 Python 再输出 .tex。

按此 prompt + 你的数据说明，即可在其他领域论文中复现同一套制表标准与逻辑。
